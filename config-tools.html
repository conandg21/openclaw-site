⚙️ tools { }

Tool Permissions Configuration

The tools block controls exactly what your agent is allowed to do — which built-in tools it can use, which skills are active, and what requires confirmation.

Complete Example

Full Tools Block

Here's a fully annotated tools configuration. Each field is explained in detail below.

openclaw.json
{
  "tools": {
    "enabled": [
      "web_search",
      "read_file",
      "write_file",
      "run_shell",
      "send_email",
      "calendar",
      "browser",
      "memory"
    ],
    "disabled": [],
    // Explicitly disable specific tools
    "requireConfirmation": [
      "write_file",
      "run_shell",
      "send_email"
    ],
    // Ask user before running these
    "allowShell": false,
    // Master switch for shell execution
    "allowFileWrite": true,
    // Master switch for file writes
    "allowNetworkRequests": true,
    // Allow outbound HTTP requests
    "maxShellTimeout": 30,
    // Max seconds for shell commands
    "sandboxed": false,
    // Run tools in isolated environment
    "skills": {
      "enabled": true,
      "directory": "~/.openclaw/skills",
      // Where skill files are stored
      "autoload": true
      // Load all skills at startup
    },
    "mcp": {
      "enabled": true
      // Enable MCP server tool calls
    }
  }
}
💡

Security Recommendation: Always set requireConfirmation to ["write_file", "run_shell", "send_email"] in production. These tools can modify your system or send messages.

Reference

Field-by-Field Configuration

Field Type Default Description
enabled array all List of tool names that are available to the agent. If empty, no tools are enabled.
disabled array [] Explicitly disable specific tools by name. Takes precedence over enabled.
requireConfirmation array [] Tools that require user approval before execution. Agent will prompt user in chat.
allowShell boolean false Master switch for shell command execution. If false, run_shell is disabled.
allowFileWrite boolean true Master switch for file write operations. If false, write_file is disabled.
allowNetworkRequests boolean true Allow outbound HTTP requests (web search, API calls). If false, network tools are disabled.
maxShellTimeout number 30 Maximum seconds a shell command can run before timing out. Prevents runaway processes.
sandboxed boolean false Run all tools in an isolated sandbox environment. Limits filesystem and network access.
skills.enabled boolean true Enable or disable the skills system entirely.
skills.directory string ~/.openclaw/skills Directory where skill files are stored. Supports ~ for home directory.
skills.autoload boolean true Automatically load all skills from directory at startup. If false, use active array to load specific skills.
mcp.enabled boolean true Enable MCP (Model Context Protocol) server connections and tool calls.
Available Tools

Built-in Tool Reference

OpenClaw includes these built-in tools by default. Each has a risk level that affects how you should configure it.

Tool Name What It Does Risk Notes
web_search Search the internet and fetch web pages Low Read-only. No local impact.
read_file Read files from your filesystem Low Read-only. Agent can only read, not modify.
write_file Write or edit files on disk Medium Can create, overwrite, or delete files. Add to requireConfirmation.
run_shell Execute shell commands High Full system access. Set allowShell: false by default. Add to requireConfirmation.
send_email Send email via configured account Medium Can send emails on your behalf. Add to requireConfirmation.
calendar Read and write calendar events Medium Requires calendar integration. Check permissions.
browser Control a web browser (click, type, navigate) Medium Can interact with any website. Requires care with sensitive sites.
memory Read and write agent memory Low Internal only. Stores agent state between sessions.
screenshot Capture screen contents Low Read-only. Captures current screen state.
clipboard Read and write system clipboard Low Can access sensitive data in clipboard if present.
notifications Send system notifications Low Desktop notifications only. No system access.
contacts Access system contacts Medium Requires permission on macOS. Check settings.
⚠️

Rule of thumb: Tools marked as High risk should be disabled by default or put in requireConfirmation. Tools marked Medium should be in requireConfirmation if they touch sensitive data.

Permissions

Three Permission Levels

Every tool can run at one of three permission levels. Control the level by which array the tool appears in.

✅

Auto (Default)

Tool runs without asking. Agent just executes the tool and continues. Best for read-only tools and safe operations.

🤔

Confirm

Agent asks the user before running. A prompt appears in chat: "Can I do this?" User must type "yes" or click approve. Best for write/send operations.

🚫

Disabled

Tool is never available. Agent cannot use it, even if it tries. Best for tools you don't trust or don't need.

How to Set Permission Levels

Each array controls where a tool lands on the permission spectrum:

Permission Level Examples
"enabled": [
  "web_search",      // ✅ Auto - no confirmation needed
  "read_file",       // ✅ Auto
  "write_file",      // will be overridden by requireConfirmation
  "send_email"       // will be overridden by requireConfirmation
],

"requireConfirmation": [
  "write_file",      // 🤔 Confirm - ask user first
  "run_shell",      // 🤔 Confirm
  "send_email"       // 🤔 Confirm
],

"disabled": [
  "calendar",       // 🚫 Disabled - agent can't use it
  "contacts"        // 🚫 Disabled
]
  • 💡
    Same tool, different levels: A tool can only be in ONE place. If it's in both enabled and requireConfirmation, confirmation wins.
  • 🛡️
    Recommended for production: Set requireConfirmation: ["write_file", "run_shell", "send_email"] to catch risky operations.
  • ⚡
    For maximum safety: Set allowShell: false and disabled: ["run_shell"] together. Belt and suspenders.
Examples

What Confirmation Looks Like

When a tool is in requireConfirmation, the agent asks for permission before proceeding. Here's what you'll see:

Shell Command Confirmation
Agent: I need to run a command to list your backup files. Can I execute:
ls -la ~/backups/ | head -20
User input required: Type "yes" or "allow" to continue.
File Write Confirmation
Agent: I want to update your config file. This will:
1. Set api_key to your OpenAI key
2. Enable voice mode
3. Save to ~/.openclaw/config.json
Type "yes" to confirm or "no" to skip.
⚡

No auto-timeout: Confirmation prompts wait indefinitely for user input. Agent does not proceed unless you explicitly approve. This protects you from accidental or malicious operations.

Custom Tools

Skills Configuration

Skills are custom tools you create or install. The skills block controls how they load and which ones are active.

Full Skills Configuration
"skills": {
  "enabled": true,
  // Enable/disable skills system
  "directory": "~/.openclaw/skills",
  // Where skill files live
  "autoload": true,
  // Load all .skill files
  "active": [
    "morning_briefing",
    "email_summarizer",
    "slack_bot"
  ]
  // Only load these if autoload: false
}
📦

skills.enabled

Turn the entire skills system on or off. When false, no custom skills load, only built-in tools available.

📂

skills.directory

Where OpenClaw looks for skill files. Supports tilde expansion (~). Default is ~/.openclaw/skills.

⚡

skills.autoload

If true, load all .skill files in directory. If false, only load skills listed in active array.

🎯

skills.active

List specific skills to load when autoload: false. Ignored if autoload is true. Use to control which skills are available.

  • ✅
    Autoload mode: Set autoload: true to load all skills from your directory. Good for development or when you trust all skills.
  • 🛡️
    Selective mode: Set autoload: false and list skills in active array to hand-pick which skills load. Better for production.
  • ❌
    Disable all skills: Set enabled: false to turn off the skills system entirely. Agent will only have built-in tools.
Recipes

Common Configuration Patterns

Development / Full Power

All tools enabled for rapid iteration. Use when you trust your setup.

"tools": {
  "enabled": ["web_search", "read_file", "write_file", "run_shell", "send_email"],
  "requireConfirmation": [],
  "allowShell": true,
  "allowFileWrite": true
}

Production / Safe Mode

Ask before write/shell/email. Master switch disables shell by default.

"tools": {
  "enabled": ["web_search", "read_file", "write_file", "send_email"],
  "requireConfirmation": ["write_file", "send_email"],
  "allowShell": false,
  "allowFileWrite": true
}

Read-Only / Maximum Safety

Only search and read. No file write, no shell, no email. Great for public chatbots.

"tools": {
  "enabled": ["web_search", "read_file"],
  "disabled": ["write_file", "run_shell", "send_email"],
  "allowShell": false,
  "allowFileWrite": false
}
