⚙️ gateway { }

Gateway Configuration

The gateway block controls how users talk to your agent — via Telegram, WhatsApp, Slack, Discord, iMessage, or the built-in web UI.

Full Configuration

The Complete gateway Block

Copy this into your openclaw.json and fill in the API tokens and configuration for each channel you want to enable.

{
  "gateway": {
    "telegram": {
      "enabled":       true,
      "token":         "${TELEGRAM_BOT_TOKEN}",
      "allowedUsers":  [123456789],           // Telegram user IDs (whitelist)
      "allowedGroups": [],                    // Group chat IDs (empty = no groups)
      "parseMode":     "Markdown",            // "Markdown" | "HTML" | "plain"
      "typingIndicator": true                 // Show "typing…" while agent works
    },
    "whatsapp": {
      "enabled":       false,
      "provider":      "meta-cloud",          // "meta-cloud" | "twilio"
      "phoneId":       "${WA_PHONE_NUMBER_ID}",
      "token":         "${WA_TOKEN}",
      "webhookSecret": "${WA_WEBHOOK_SECRET}",
      "webhookPath":   "/webhook/whatsapp"
    },
    "slack": {
      "enabled":       false,
      "botToken":      "${SLACK_BOT_TOKEN}",
      "signingSecret": "${SLACK_SIGNING_SECRET}",
      "allowedChannels": ["#general"],        // Empty = all channels
      "mentionOnly":   true                   // Only respond when @mentioned
    },
    "discord": {
      "enabled":       false,
      "token":         "${DISCORD_TOKEN}",
      "allowedGuilds": [],
      "allowedChannels": [],
      "prefix":        "!",                   // Command prefix (or use slash commands)
      "slashCommands": true
    },
    "web": {
      "enabled":       true,
      "port":          7799,
      "host":          "localhost",           // "0.0.0.0" to expose on network
      "auth":          false,                 // Enable basic auth for web UI
      "username":      "",
      "password":      ""
    }
  }
}

Telegram

The easiest gateway to set up. Uses long-polling so no public URL needed. Perfect for personal use and testing.

Field Type Default Status Description
enabled boolean true optional Enable or disable the Telegram gateway.
token string — required Your Telegram bot token from BotFather. Use ${TELEGRAM_BOT_TOKEN} to load from env.
allowedUsers number[] [] optional List of Telegram user IDs allowed to use the bot. Empty means anyone can. Whitelist by ID for security.
allowedGroups number[] [] optional List of group chat IDs allowed to use the bot. Empty means no groups. Get IDs by using /getchatid in a group.
parseMode string "Markdown" optional How to format text responses: "Markdown", "HTML", or "plain". Markdown is most portable.
typingIndicator boolean true optional Show "typing…" status while the agent is processing. Good for UX.

How to Get Your User ID

To find your Telegram user ID, add this bot to your chat and send any message: @userinfobot. It will reply with your numeric ID. Add that ID to the allowedUsers array.

WhatsApp

Requires a Meta Business account and a public webhook URL. Most complex to set up, but reaches the broadest audience.

Field Type Default Status Description
enabled boolean false optional Enable or disable the WhatsApp gateway.
provider string — required WhatsApp API provider: "meta-cloud" (Meta's official Cloud API) or "twilio" (Twilio). Meta is recommended.
phoneId string — required Your WhatsApp business phone number ID. Found in Meta Business Manager under Phone Numbers.
token string — required Meta Cloud API access token. Use ${WA_TOKEN} to load from env.
webhookSecret string — required Webhook verify token for signature validation. Use ${WA_WEBHOOK_SECRET} to load from env.
webhookPath string "/webhook/whatsapp" optional Path where Meta sends webhook events. Must match your Meta app config.

Slack

Best for team communication and internal use. Requires a public webhook URL but is straightforward to set up with any Slack workspace.

Field Type Default Status Description
enabled boolean false optional Enable or disable the Slack gateway.
botToken string — required OAuth token for your Slack bot app (starts with xoxb-). Use ${SLACK_BOT_TOKEN} to load from env.
signingSecret string — required Signing secret from your Slack app settings for request verification. Use ${SLACK_SIGNING_SECRET} to load from env.
allowedChannels string[] [] optional List of channel names (with #) the bot can respond in. Empty array means all channels.
mentionOnly boolean true optional If true, bot only responds when @mentioned. If false, responds to all messages in allowed channels.

Discord

Great for communities and gaming servers. Uses websockets so no public URL required. Supports slash commands.

Field Type Default Status Description
enabled boolean false optional Enable or disable the Discord gateway.
token string — required Discord bot token from the Developer Portal. Use ${DISCORD_TOKEN} to load from env. Keep this secret!
allowedGuilds string[] [] optional List of Discord guild (server) IDs allowed to use the bot. Empty means all guilds your bot is in.
allowedChannels string[] [] optional List of Discord channel IDs allowed to use the bot. Empty means all channels in allowed guilds.
prefix string "!" optional Prefix for text commands (e.g., !ask). Only used if slash commands are disabled.
slashCommands boolean true optional Enable Discord slash commands. Recommended over prefix commands.

Web UI

Built-in web interface. Runs locally by default. No authentication needed for localhost, but can add basic auth if exposed to the network.

Field Type Default Status Description
enabled boolean true optional Enable or disable the web UI server.
port number 7799 optional Port the web server listens on. Access at http://localhost:7799.
host string "localhost" optional Host to bind to. Use "0.0.0.0" to expose on your network. Warning: Only do this with auth enabled.
auth boolean false optional Enable HTTP Basic Authentication. Required if exposing to network with "0.0.0.0".
username string "" optional Basic auth username. Only used if auth: true.
password string "" optional Basic auth password. Only used if auth: true. Use env vars like ${WEB_PASSWORD}.
Quick Comparison

Choosing a Gateway

Not sure which channel is right for your use case? Here's a quick breakdown of setup difficulty, infrastructure needs, and best use cases.

Channel Setup Difficulty Public URL? Mobile Desktop Best For
Telegram Easy No Yes Yes Personal use, testing, quick setup
WhatsApp Hard Yes Yes Yes Mass reach, international, business
Slack Medium Yes Yes Yes Team workflows, internal automation
Discord Medium No Yes Yes Communities, gaming, open discussion
Web UI None No Browser Browser Built-in, localhost-only by default
Infrastructure

Webhooks vs Polling

Different channels use different connection patterns. Understanding this matters for deployment.

ℹ️

Telegram uses long-polling: Your OpenClaw instance continually asks Telegram for new messages. No public URL needed. Simple, but slightly higher latency and network overhead.

ℹ️

WhatsApp & Slack use webhooks: Meta and Slack send incoming messages directly to a publicly accessible URL on your server. Much lower latency, but requires your server to be internet-facing with HTTPS.

ℹ️

Discord uses websockets: Your bot maintains a persistent websocket connection to Discord. Low latency, no public URL required, but your bot must be online at all times.

If You Need a Public URL

For WhatsApp and Slack, expose your OpenClaw server publicly:

  • ngrok: Free tool for tunneling. Run ngrok http 7799 to get a public URL for localhost testing.
  • VPS (DigitalOcean, AWS, etc.): Deploy OpenClaw on a server with a static IP and domain name.
  • Your own domain: If you own a domain, use DNS to point it to your OpenClaw instance and add HTTPS with Let's Encrypt.
Security Best Practices

Protecting Your Agent with Allowlists

Without user and channel allowlists, anyone who finds your bot token can use your agent. Here's why and how to restrict access.

Telegram: allowedUsers and allowedGroups

If you don't set allowedUsers, anyone on Telegram can message your bot and trigger your agent. If your agent can make API calls or access databases, this is a security risk.

{
  "telegram": {
    "enabled": true,
    "token": "${TELEGRAM_BOT_TOKEN}",
    "allowedUsers": [123456789, 987654321],  // Only these users can use the bot
    "allowedGroups": [-1001234567890]        // Only this group can use the bot
  }
}

Slack: allowedChannels and mentionOnly

By default, if mentionOnly is true, the bot ignores messages where it's not mentioned. For extra control, specify allowedChannels to restrict to specific channels:

{
  "slack": {
    "enabled": true,
    "botToken": "${SLACK_BOT_TOKEN}",
    "signingSecret": "${SLACK_SIGNING_SECRET}",
    "allowedChannels": ["#ai-team", "#beta-testers"],
    "mentionOnly": true
  }
}

Discord: allowedGuilds and allowedChannels

Restrict your Discord bot to specific servers and channels:

{
  "discord": {
    "enabled": true,
    "token": "${DISCORD_TOKEN}",
    "allowedGuilds": ["1234567890"],        // Only this guild
    "allowedChannels": ["9876543210"]      // Only this channel
  }
}

Environment Variables for Secrets

Never hardcode API tokens in your config file. Always use environment variables:

// ✅ Good: uses env var
"token": "${TELEGRAM_BOT_TOKEN}"

// ❌ Bad: hardcoded secret
"token": "123456:ABCdefGHIjklMNOpqrsTUVwxyz"
