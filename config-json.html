Every key your agent understands, documented in one place. Find your config file, understand each setting, and copy ready-to-use snippets.

Where is the File?

By default, OpenClaw looks for openclaw.json in your home directory at ~/.openclaw/openclaw.json.

Override the location: Set the OPENCLAW_CONFIG environment variable to point to a different path:

export OPENCLAW_CONFIG=/path/to/config.json
openclaw

Quick commands:

  • openclaw config edit — Open your config in your default editor
  • openclaw config validate — Check your config for errors before restarting
  • openclaw config reload — Reload without restarting (for quick changes)

Field Reference by Section

llm — Language Model Configuration

The LLM section defines which AI model powers your agent and how it behaves.

{
  "llm": {
    "provider": "anthropic",          // Required: "anthropic" | "openai" | "google" | "ollama"
    "model": "claude-3-5-sonnet-20241022",  // Required: model ID
    "apiKey": "${ANTHROPIC_API_KEY}", // Required (unless using local Ollama)
    "baseUrl": "https://api.anthropic.com", // Optional: override endpoint
    "temperature": 0.7,               // 0.0–2.0, lower = more focused, default 0.7
    "maxTokens": 4096,                // Max tokens in response, default 4096
    "timeout": 30000,                 // Request timeout in ms, default 30000
    "retries": 2,                     // Retry failed requests, default 2
    "cache": true,                    // Enable prompt caching, default true
    "heartbeatModel": "claude-3-5-haiku-20241022",  // Optional: lightweight model for heartbeat checks
    "powerModel": "claude-3-5-sonnet-20241022"      // Optional: more capable model for complex tasks
  }
}
Key Type Default Required Description
provider string — Yes AI provider: "anthropic", "openai", "google", or "ollama"
model string — Yes Model ID (e.g. "claude-3-5-sonnet-20241022" for Anthropic)
apiKey string — Yes* API key. Use "${VAR}" syntax for environment variables. Not needed for local Ollama
baseUrl string Provider default No Override API endpoint (e.g., for Ollama on localhost:11434)
temperature number 0.7 No Randomness: 0 = deterministic, 2.0 = creative
maxTokens number 4096 No Maximum tokens per response
timeout number 30000 No Request timeout in milliseconds
retries number 2 No Number of retries on failure
cache boolean true No Enable prompt caching (saves API cost)
heartbeatModel string — No Lightweight model for periodic checks. Uses "model" if omitted
powerModel string — No More capable model for complex reasoning. Uses "model" if omitted

gateway — Integration Channels

Enable integrations with chat platforms (Slack, Discord, Telegram, WhatsApp) and web interface.

{
  "gateway": {
    "web": true,                      // Enable web UI, default true
    "webPort": 7799,                  // Port for web interface, default 7799
    "webHost": "localhost",           // Host binding, default "localhost"
    "telegram": false,                // Enable Telegram, default false
    "telegramToken": "${TELEGRAM_TOKEN}",
    "telegramAllowedUsers": [123456789, 987654321],
    "whatsapp": false,                // Enable WhatsApp, default false
    "whatsappToken": "${WHATSAPP_TOKEN}",
    "whatsappPhoneId": "1234567890",
    "whatsappWebhookSecret": "${WHATSAPP_SECRET}",
    "slack": false,                   // Enable Slack, default false
    "slackBotToken": "${SLACK_BOT_TOKEN}",
    "slackAppToken": "${SLACK_APP_TOKEN}",
    "slackAllowedChannels": ["general", "dev"],
    "discord": false,                 // Enable Discord, default false
    "discordToken": "${DISCORD_TOKEN}",
    "discordAllowedGuilds": [9876543210, 1234567890]
  }
}
Key Type Default Required Description
web boolean true No Enable web interface at localhost:webPort
webPort number 7799 No Port for web server
webHost string localhost No Host to bind to (use "0.0.0.0" for remote access)
telegram boolean false No Enable Telegram integration
telegramToken string — If telegram=true Telegram bot token from BotFather
telegramAllowedUsers number[] [] No Array of user IDs permitted to message the bot
whatsapp boolean false No Enable WhatsApp integration
whatsappToken string — If whatsapp=true WhatsApp API token
whatsappPhoneId string — If whatsapp=true Your WhatsApp Business phone ID
whatsappWebhookSecret string — If whatsapp=true Webhook secret for incoming WhatsApp messages
slack boolean false No Enable Slack integration
slackBotToken string — If slack=true Bot token (xoxb-...)
slackAppToken string — If slack=true App-level token (xapp-...)
slackAllowedChannels string[] [] No Channel names permitted to message the bot
discord boolean false No Enable Discord integration
discordToken string — If discord=true Discord bot token
discordAllowedGuilds number[] [] No Guild IDs permitted to use the bot

memory — Persistent Memory & Knowledge

Configure where and how your agent remembers conversations and learns from past interactions.

{
  "memory": {
    "directory": "~/.openclaw/memory",   // Location of memory files, default "~/.openclaw/memory"
    "memoryFile": "MEMORY.md",           // Filename for persistent memory, default "MEMORY.md"
    "dailyNotes": true,                  // Create daily summary notes, default true
    "retentionDays": 90,                 // Keep memory for N days (0 = forever), default 90
    "maxMemoryWords": 2000,              // Max words in active memory, default 2000
    "autoSummarise": false,              // Auto-compress old memories, default false
    "search": {
      "enabled": true,                   // Enable memory search, default true
      "fuzzy": true,                     // Fuzzy matching on queries, default true
      "maxResults": 10                   // Max search results, default 10
    }
  }
}
Key Type Default Required Description
directory string ~/.openclaw/memory No Directory to store memory files (~ expands to home)
memoryFile string MEMORY.md No Main memory file name
dailyNotes boolean true No Create dated summary files (YYYY-MM-DD.md)
retentionDays number 90 No Auto-delete memories older than N days. Set to 0 to keep forever
maxMemoryWords number 2000 No Soft limit on active memory context size
autoSummarise boolean false No Automatically compress and summarize old memories
search.enabled boolean true No Enable keyword search across memory
search.fuzzy boolean true No Allow typo-tolerant fuzzy search
search.maxResults number 10 No Return top N matching memories

tools — Permission & Security Controls

Define which tools your agent can use and under what conditions. Restrict file access and block dangerous domains.

{
  "tools": {
    "shell": "confirm",               // "auto"|"confirm"|"disabled"
    "fileRead": "auto",               // Read files without prompting
    "fileWrite": "confirm",           // Always ask before writing
    "browser": "confirm",             // Open browser / visit URLs
    "email": "confirm",               // Send emails
    "calendar": "confirm",            // Access calendar
    "http": "confirm",                // Make HTTP requests
    "code": "confirm",                // Execute code
    "allowedPaths": [
      "${HOME}/projects",
      "/tmp"
    ],
    "blockedDomains": [
      "malicious.com",
      "phishing.net"
    ]
  }
}
Key Type Default Required Description
shell string confirm No "auto" = run shell commands without asking, "confirm" = ask first, "disabled" = never run
fileRead string auto No Permission level for reading files
fileWrite string confirm No Permission level for writing/modifying files
browser string confirm No Permission to open browser and visit URLs
email string confirm No Permission to send emails
calendar string confirm No Permission to access calendar
http string confirm No Permission to make HTTP/API requests
code string confirm No Permission to execute code (Python, Node, etc.)
allowedPaths string[] [] No Whitelist of directories agent can access. Empty = all allowed
blockedDomains string[] [] No Domains to block from HTTP requests

voice — Speech & Audio Configuration

Enable voice interaction with speech-to-text (STT) and text-to-speech (TTS).

{
  "voice": {
    "stt": {
      "enabled": false,               // Enable speech-to-text
      "provider": "openai",           // "openai" | "google" | "azure"
      "apiKey": "${OPENAI_API_KEY}",
      "language": "en"                // Language code, default "en"
    },
    "tts": {
      "enabled": false,               // Enable text-to-speech
      "provider": "elevenlabs",       // "elevenlabs" | "native" | "google"
      "apiKey": "${ELEVENLABS_API_KEY}",
      "voiceId": "default",           // Voice ID from provider
      "model": "eleven_monolingual_v1"
    },
    "wakeWord": {
      "enabled": false,               // Activate on wake phrase
      "phrases": ["hey openclaw"]      // List of wake phrases
    },
    "silenceTimeoutMs": 1500,         // Stop listening after 1.5s silence
    "interruptOnSpeech": true,        // Stop playing TTS if user speaks
    "quietHours": {
      "start": "22:00",               // No voice alerts after 10 PM
      "end": "08:00"                  // Resume at 8 AM
    }
  }
}
Key Type Default Required Description
stt.enabled boolean false No Enable speech recognition
stt.provider string openai No STT service provider
stt.apiKey string — If stt.enabled=true API key for STT provider
stt.language string en No Language code (e.g., "en", "es", "fr")
tts.enabled boolean false No Enable text-to-speech output
tts.provider string elevenlabs No TTS service: "elevenlabs", "native", or "google"
tts.apiKey string — If tts.enabled=true API key for TTS provider
tts.voiceId string default No Voice ID from TTS provider
tts.model string — No TTS model name (e.g., "eleven_monolingual_v1")
wakeWord.enabled boolean false No Activate only on wake phrase
wakeWord.phrases string[] [] No Wake phrases to listen for
silenceTimeoutMs number 1500 No Milliseconds of silence before stopping listening
interruptOnSpeech boolean true No Stop TTS output when user speaks
quietHours.start string — No Time in HH:MM format (24-hour) to stop voice alerts
quietHours.end string — No Time in HH:MM format (24-hour) to resume voice alerts

heartbeat — Periodic Check-ins

Configure how often your agent checks for new tasks and proactively reaches out.

{
  "heartbeat": {
    "enabled": true,                  // Enable periodic check-ins, default true
    "interval": 1800,                 // Check every 1800 seconds (30 min), default 1800
    "quietHours": {
      "start": "22:00",               // Don't check after 10 PM
      "end": "08:00"                  // Resume at 8 AM
    }
  }
}
Key Type Default Required Description
enabled boolean true No Enable periodic check-ins
interval number 1800 No Seconds between checks
quietHours.start string — No Time in HH:MM to start quiet period
quietHours.end string — No Time in HH:MM to end quiet period

mcp — Model Context Protocol Servers

Connect external tools and data sources via MCP servers.

{
  "mcp": {
    "mcpServers": {
      "filesystem": {                 // Server name (any label)
        "command": "node",            // Command to start server
        "args": [                     // Arguments to pass
          "mcp-server-filesystem.js"
        ],
        "env": {                      // Optional environment variables
          "DEBUG": "false"
        }
      },
      "postgresql": {
        "command": "python",
        "args": ["-m", "mcp_server_postgres"],
        "env": {
          "DATABASE_URL": "${DATABASE_URL}"
        }
      }
    }
  }
}
Key Type Default Required Description
mcpServers object {} No Object mapping server names to configuration
mcpServers[name].command string — Yes Executable to run (e.g., "node", "python", "bash")
mcpServers[name].args string[] [] No Arguments passed to the command
mcpServers[name].env object {} No Environment variables for the server (supports ${VAR} substitution)

logging — Debug & Diagnostics

Configure logging level and output for troubleshooting.

{
  "logging": {
    "level": "info",                  // "debug"|"info"|"warn"|"error", default "info"
    "file": "~/.openclaw/logs/openclaw.log",  // Optional: write logs to file
    "maxSizeMb": 10,                  // Rotate log file at 10 MB, default 10
    "rotate": true                    // Enable log rotation, default true
  }
}
Key Type Default Required Description
level string info No "debug" = verbose, "info" = normal, "warn" = warnings only, "error" = errors only
file string — No Path to log file. If omitted, logs to console only
maxSizeMb number 10 No Max log file size before rotation
rotate boolean true No Enable automatic log rotation

Pro Tips

💾 Use Environment Variables

Never hardcode secrets. Use "${VAR_NAME}" syntax to reference environment variables, and OpenClaw will substitute them at startup.

✅ Validate Before Restarting

Run openclaw config validate to catch syntax errors and missing required fields before you restart your agent.

📝 JSON Doesn't Support Comments

Add "_comment": "your note here" keys instead, or use a JSON with comments (JSONC) editor. Strip comments before deployment.

🚀 Start Minimal

Only llm is required. Add other sections (gateway, voice, mcp) as you need them. Start simple and expand.
