Getting Started

Where Is Your Config File?

OpenClaw looks for your configuration file in predictable places. You can override the default location if you need to.

📍

Default location: ~/.openclaw/openclaw.json — OpenClaw checks here first on every startup.

Environment Variable

Override the default path with:

OPENCLAW_CONFIG=/path/to/config.json

CLI Flag

Pass the config path at launch:

openclaw --config /path/to/config.json

Quick Edit

Open in your default editor:

openclaw config edit
The Big Picture

Full openclaw.json Structure

Here's every top-level key in your configuration file. Each one controls a major subsystem.

{
  "llm": { /* → Model provider, API key, model name, temperature */ },
  "gateway": { /* → Messaging channels: Telegram, WhatsApp, Slack, Discord */ },
  "tools": { /* → Which tools/skills the agent can use, permissions */ },
  "memory": { /* → Memory storage, retention, search, daily notes */ },
  "voice": { /* → STT provider, TTS provider, wake word detection */ },
  "heartbeat": { /* → Scheduled background tasks, cron jobs */ },
  "mcp": { /* → MCP server connections and configuration */ },
  "logging": { /* → Log level, output path, rotation */ }
}

Important

Configuration Format & Tips

  • 📄
    JSON only — no YAML OpenClaw uses strict JSON format. YAML, TOML, and other formats are not supported in current versions. Remember to quote all keys and values properly.
  • ✅
    Comments are not allowed Standard JSON does not support comments. If you need to leave notes, create a separate `_notes.txt` file. Use openclaw config validate to check for syntax errors.
  • ⚡
    Reload without restarting After editing your config, run openclaw config reload to apply changes immediately. You don't need to stop and restart the agent for most settings.
  • 🔐
    Environment variable interpolation All string values support env var expansion using "${VAR_NAME}" syntax. Example: "apiKey": "${OPENAI_API_KEY}". This keeps secrets out of your config file.
