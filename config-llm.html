⚙️ llm { }

LLM & Model Configuration

The llm block controls which AI model OpenClaw uses, how it authenticates, and how it routes different types of requests.

Config Reference → llm { }
Full Configuration

Complete LLM Block

Here's a complete llm block with all available fields, inline comments, and sensible defaults:

{
  "llm": {
    "provider":        "anthropic",       // "anthropic" | "openai" | "google" | "ollama" | "groq"
    "model":           "claude-haiku-4",  // Model name from your provider
    "apiKey":          "${ANTHROPIC_API_KEY}",  // Use env vars — never hardcode keys
    "baseUrl":         "",                // Override API endpoint (for Ollama, proxies)
    "temperature":     0.7,               // 0.0 (deterministic) – 1.0 (creative)
    "maxTokens":       4096,              // Max output tokens per response
    "contextWindow":   200000,            // Max input tokens (set to your model's limit)
    "heartbeatModel":  "ollama/llama3.2", // Cheaper model for background tasks
    "powerModel":      "claude-sonnet-4", // Used when agent requests heavy reasoning
    "cache":           true,              // Enable prompt caching (reduces costs ~80%)
    "timeout":         30,                // Request timeout in seconds
    "retries":         3                  // Auto-retry on API errors
  }
}
💡

Environment Variables: Use ${VAR_NAME} syntax in any string field to load values from your .env file. Never hardcode API keys in openclaw.json.

Field Reference

All LLM Fields Explained

Field Type Default Required Description
provider string "anthropic" Yes Which LLM API to use: anthropic, openai, google, ollama, or groq.
model string — Yes The specific model to use. Must match your provider's available models. See provider setup cards below.
apiKey string — No* API key for your provider. Use ${ENV_VAR} syntax. Not needed for Ollama (local). Treated as secret — never logged or exposed.
baseUrl string "" No Override the default API endpoint. Use for self-hosted Ollama servers, proxy URLs, or custom API gateways. Example: http://localhost:11434
temperature number 0.7 No Controls randomness: 0.0 = deterministic/factual, 1.0 = creative/varied. Range: 0.0 – 1.0. Recommended: 0.7 for general use.
maxTokens number 4096 No Maximum tokens in a single response. Lower values save costs; higher values allow longer outputs. Typical range: 1024 – 8192.
contextWindow number 200000 No Maximum input tokens the model can accept. Set this to your model's limit to avoid token count errors. Examples: Claude Haiku = 200k, GPT-4o = 128k, Ollama = varies.
heartbeatModel string — No Cheaper/faster model for background tasks defined in HEARTBEAT.md. Saves cost on routine operations. Can use a different provider (e.g., ollama/llama3.2).
powerModel string — No Premium model used when AGENTS.md specifies use_power_model: true. For heavy reasoning tasks. Can use a different provider.
cache boolean true No Enable prompt caching (Anthropic). Repeated inputs are cached, reducing API costs by ~80% on subsequent requests. Requires provider support.
timeout number 30 No Request timeout in seconds. If the API doesn't respond within this time, the request is aborted. Typical: 15–60 seconds.
retries number 3 No Number of automatic retries on transient API errors (timeouts, rate limits, 5xx errors). Typical: 2–5.

* Ollama does not require an API key; all other providers do.

Advanced

Multi-Model Routing

Use three different models for different types of tasks — optimizing for cost and capability.

OpenClaw supports routing requests to different models based on the task type:

model

Default for all conversations and general-purpose requests.

Use case: User chat, questions, brainstorming.

heartbeatModel

Used for background tasks defined in HEARTBEAT.md. Usually cheaper/faster.

Use case: Scheduled tasks, cleanup, indexing.

powerModel

Used when AGENTS.md specifies use_power_model: true. Premium capability.

Use case: Complex reasoning, code generation, analysis.

// Example configuration with model routing
{
  "llm": {
    "provider": "anthropic",
    "model": "claude-haiku-4",        // Fast, cheap — default for chat
    "heartbeatModel": "ollama/llama3.2",  // Local, free — background tasks
    "powerModel": "claude-opus-4"     // Most capable — heavy reasoning
  }
}

How the agent switches models: When an agent in AGENTS.md sets use_power_model: true, OpenClaw automatically routes that agent's requests to the powerModel instead of the default model. Heartbeat tasks always use heartbeatModel regardless of agent config.

Settings Guide

Temperature: Determinism vs. Creativity

Temperature controls how "random" or "creative" the model's responses are. Lower = more predictable, higher = more varied.

Temperature Behavior Use Case
0.0 Fully deterministic. Exact same output every time. Factual Q&A, data extraction, structured output.
0.3 Low randomness. Mostly consistent, slight variation. Technical writing, code generation, instructions.
0.7 Balanced. Natural, varied responses. Recommended default. General chat, brainstorming, creative tasks.
1.0 Maximum randomness. Highly creative, unpredictable. Creative writing, ideation, storytelling.
🎯

For most OpenClaw deployments, 0.7 is ideal. It balances natural conversation with consistency. Increase to 0.8–1.0 for creative tasks; decrease to 0.3–0.5 for factual work.

Configuration

Environment Variable Interpolation

Store sensitive data outside of openclaw.json using environment variables.

OpenClaw loads variables from a .env file in your project root. Use the ${VAR_NAME} syntax in any field to reference them:

openclaw.json

{
  "llm": {
    "provider": "anthropic",
    "apiKey": "${ANTHROPIC_API_KEY}",
    "baseUrl": "${OLLAMA_URL}"
  }
}

.env (in project root)

ANTHROPIC_API_KEY=sk-ant-v0-abc123xyz
OLLAMA_URL=http://localhost:11434
OPENAI_API_KEY=sk-proj-abc123
GOOGLE_API_KEY=AIzaSyABC123
🔒

Important: Add .env to your .gitignore file so API keys are never committed to version control. Example: echo ".env" >> .gitignore

How it works: When OpenClaw starts, it reads .env and replaces all ${VAR_NAME} references with their actual values. This works in any string field, including apiKey, baseUrl, and more.
