ARCHITECTURE

Scanner Architecture

The multi-stock scanner follows a simple but powerful pipeline:

  • Input: List of tickers (your watchlist or full S&P 500)
  • Fetch: Download 1 year of daily OHLCV (Open, High, Low, Close, Volume) for each ticker
  • Compute: Calculate all technical indicators (SMA, RSI, Bollinger Bands, MACD) on the fetched data
  • Evaluate: Check signal conditions (RSI extremes, moving average crosses, band squeezes, MACD crossovers)
  • Rank: Sort results by signal count and RSI extremity
  • Output: Display results in a terminal table and export to CSV

Key principle: Fetch once, evaluate multiple conditions. Cache data to avoid redundant downloads and speed up re-runs on the same day.

CORE LOGIC

Core Scanner Function

This function handles a single ticker: fetch data, compute all indicators, and evaluate signal conditions. It returns a result dictionary containing price, RSI, Bollinger Bands width, and all detected signals.

python 
import yfinance as yf
import pandas_ta as ta
import time
from datetime import datetime

def scan_ticker(ticker: str, period: str = "1y") -> dict | None:
    """
    Fetch data and evaluate all signal conditions for one ticker.
    Returns a result dict, or None on error.
    """
    try:
        df = yf.download(ticker, period=period, interval="1d", progress=False)
        if df.empty or len(df) < 210:
            return None
        df.columns = [c.lower() for c in df.columns]
        df = df[["open","high","low","close","volume"]].dropna()

        # Compute indicators
        df["sma50"]  = ta.sma(df["close"], 50)
        df["sma200"] = ta.sma(df["close"], 200)
        df["rsi"]    = ta.rsi(df["close"], 14)
        bb = ta.bbands(df["close"], 20, 2)
        df["bb_bw"]  = bb[[c for c in bb.columns if "BBB" in c][0]]
        df["bb_upper"] = bb[[c for c in bb.columns if "BBU" in c][0]]
        df["bb_lower"] = bb[[c for c in bb.columns if "BBL" in c][0]]
        macd = ta.macd(df["close"], 12, 26, 9)
        df["macd"]   = macd[[c for c in macd.columns if c.startswith("MACD_")][0]]
        df["macd_s"] = macd[[c for c in macd.columns if c.startswith("MACDs_")][0]]
        df = df.dropna()

        if len(df) < 2:
            return None

        cur = df.iloc[-1]
        prev = df.iloc[-2]

        # Signal evaluations
        rsi_val = cur["rsi"]
        bw_20th = df["bb_bw"].quantile(0.20)

        signals = []
        if rsi_val < 30:   signals.append("RSI_OVERSOLD")
        if rsi_val > 70:   signals.append("RSI_OVERBOUGHT")
        if (cur["sma50"] > cur["sma200"]) and (prev["sma50"] <= prev["sma200"]):
            signals.append("GOLDEN_CROSS")
        if (cur["sma50"] < cur["sma200"]) and (prev["sma50"] >= prev["sma200"]):
            signals.append("DEATH_CROSS")
        if cur["bb_bw"] < bw_20th:
            signals.append("BB_SQUEEZE")
        if (cur["macd"] > cur["macd_s"]) and (prev["macd"] <= prev["macd_s"]):
            signals.append("MACD_BULL_CROSS")
        if (cur["macd"] < cur["macd_s"]) and (prev["macd"] >= prev["macd_s"]):
            signals.append("MACD_BEAR_CROSS")

        # Above/below key MAs
        ma_status = []
        if cur["close"] > cur["sma50"]:  ma_status.append("Above50")
        if cur["close"] > cur["sma200"]: ma_status.append("Above200")

        pct_change_1d = (cur["close"] - prev["close"]) / prev["close"] * 100

        return {
            "ticker":     ticker,
            "price":      cur["close"],
            "chg_1d":     pct_change_1d,
            "rsi":        rsi_val,
            "bb_bw":      cur["bb_bw"],
            "macd_hist":  cur["macd"] - cur["macd_s"],
            "signals":    signals,
            "ma_status":  " ".join(ma_status) if ma_status else "Below both",
            "signal_count": len(signals),
        }
    except Exception as e:
        return None

The function uses pandas_ta to compute all indicators efficiently. It checks for 7 distinct signal types, tracks moving average positioning, and returns a compact dictionary for each ticker. All calculations happen on the same dataframe to minimize memory overhead.

EXECUTION

Run the Full Scanner

The run_scanner() function loops through all tickers, calls scan_ticker() on each, and collects results. It includes a small delay between requests (to be polite to Yahoo Finance) and sorts results by signal count.

python 
from tabulate import tabulate
import csv

def run_scanner(tickers: list, period: str = "1y", delay: float = 0.1) -> list:
    """
    Scan all tickers. Returns list of result dicts sorted by signal count.
    delay: seconds between requests (be polite to Yahoo Finance)
    """
    results = []
    errors = 0

    print(f"\n🔍 Scanning {len(tickers)} tickers | {datetime.now().strftime('%Y-%m-%d %H:%M')}")
    print("─" * 50)

    for i, ticker in enumerate(tickers, 1):
        result = scan_ticker(ticker, period)
        if result:
            if result["signals"]:
                results.append(result)
                print(f"  [{i:>3}/{len(tickers)}] {ticker:<8} ✓  {', '.join(result['signals'])}")
            else:
                print(f"  [{i:>3}/{len(tickers)}] {ticker:<8} —  no signals")
        else:
            errors += 1
            print(f"  [{i:>3}/{len(tickers)}] {ticker:<8} ✗  error/skip")
        time.sleep(delay)

    # Sort: most signals first, then by RSI extremity
    results.sort(key=lambda x: (x["signal_count"], abs(x["rsi"] - 50)), reverse=True)
    return results

results = run_scanner(WATCHLIST)

The scanner prints progress in real-time, showing each ticker's status and detected signals. Tickers with no signals are noted but not included in the results. With a 24-ticker watchlist, the full scan completes in 30–40 seconds. The full S&P 500 takes 2–3 minutes depending on network speed.

OUTPUT

Output: Terminal Table + CSV

After scanning, format the results as a clean terminal table and optionally save to CSV for further analysis or record-keeping.

python 
def print_scanner_results(results: list) -> None:
    """Pretty-print scanner results."""
    if not results:
        print("\nNo signals found.")
        return

    rows = []
    for r in results:
        rows.append([
            r["ticker"],
            f"${r['price']:,.2f}",
            f"{r['chg_1d']:+.1f}%",
            f"{r['rsi']:.1f}",
            r["ma_status"],
            ", ".join(r["signals"]) or "—",
        ])

    print(f"\n{'═'*70}")
    print(f"  Scanner Results — {datetime.now().strftime('%Y-%m-%d')} | {len(results)} signals found")
    print(f"{'═'*70}")
    print(tabulate(rows,
        headers=["Ticker","Price","1d%","RSI","vs MAs","Signals"],
        tablefmt="rounded_outline",
        colalign=("left","right","right","right","left","left")
    ))

def save_to_csv(results: list, path: str = None) -> None:
    """Save scanner results to CSV."""
    filename = path or f"scanner_{datetime.now().strftime('%Y%m%d')}.csv"
    if not results:
        return

    keys = ["ticker","price","chg_1d","rsi","bb_bw","ma_status","signals","signal_count"]
    with open(filename, "w", newline="") as f:
        writer = csv.DictWriter(f, fieldnames=keys, extrasaction="ignore")
        writer.writeheader()
        for r in results:
            row = dict(r)
            row["signals"] = " | ".join(row["signals"])
            writer.writerow(row)
    print(f"\nSaved: {filename}")

print_scanner_results(results)
save_to_csv(results)

The terminal table uses tabulate with rounded box borders and right-aligned numeric columns for easy scanning. The CSV export preserves all fields and joins multiple signals with a pipe separator for clean parsing later.

FILTERING

Filtering to Specific Signal Types

Once you have results, filter them to focus on a single signal type. This is useful for trading strategies that target specific patterns.

python 
def filter_signals(results: list, signal_type: str) -> list:
    """Filter results to a specific signal type."""
    return [r for r in results if signal_type in r["signals"]]

# Examples
oversold  = filter_signals(results, "RSI_OVERSOLD")
squeezes  = filter_signals(results, "BB_SQUEEZE")
gold_x    = filter_signals(results, "GOLDEN_CROSS")
macd_bull = filter_signals(results, "MACD_BULL_CROSS")

print(f"RSI Oversold:   {len(oversold)} tickers")
print(f"BB Squeezes:    {len(squeezes)} tickers")
print(f"Golden Crosses: {len(gold_x)} tickers")
print(f"MACD Bull:      {len(macd_bull)} tickers")

Each filter returns a subset of the results containing only tickers with that specific signal. You can chain filters or use them to build custom alerts for different trading strategies.

OPTIMIZATION

Performance Tips for Large Universes

When scanning the full S&P 500 repeatedly, caching is essential. Store results locally keyed by date, so re-runs on the same day are nearly instantaneous.

python 
import os
import pickle
from datetime import date

CACHE_DIR = ".scanner_cache"

def cached_scan_ticker(ticker: str, period: str = "1y") -> dict | None:
    """Cache results so re-runs don't re-fetch the same day's data."""
    os.makedirs(CACHE_DIR, exist_ok=True)
    cache_file = os.path.join(CACHE_DIR, f"{ticker}_{date.today()}.pkl")

    if os.path.exists(cache_file):
        with open(cache_file, "rb") as f:
            return pickle.load(f)

    result = scan_ticker(ticker, period)
    if result:
        with open(cache_file, "wb") as f:
            pickle.dump(result, f)
    return result

# With caching, re-running the scanner on the same day takes < 5 seconds

Each cache file stores a single ticker's scan result for the day. When you run the scanner again, it loads from cache instead of re-fetching data from Yahoo Finance. Old cache files can be safely deleted at the end of each week.

REFERENCE

Signal Types Reference

All 7 signal types detected by the scanner, their conditions, typical frequency, and trading use cases:

Signal Condition Frequency Typical Use
RSI_OVERSOLD RSI < 30 ~5–10% of universe Mean reversion watch
RSI_OVERBOUGHT RSI > 70 ~5–10% of universe Momentum or fade watch
GOLDEN_CROSS SMA50 crosses above SMA200 Rare, 1–3x/year per ticker Trend following entry
DEATH_CROSS SMA50 crosses below SMA200 Rare, 1–3x/year per ticker Trend following exit
BB_SQUEEZE Bandwidth < 20th percentile ~20% of universe at any time Breakout anticipation
MACD_BULL_CROSS MACD crosses above signal Monthly Momentum entry
MACD_BEAR_CROSS MACD crosses below signal Monthly Momentum exit

RSI and Bollinger Band signals are continuous — they update daily as prices move in and out of extremes. Moving average crosses (golden/death) and MACD crossovers are rare events that trigger only once per occurrence. Combine signals for stronger conviction.
