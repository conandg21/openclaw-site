config.yaml — Complete Field Reference

Prefer YAML over JSON? OpenClaw supports config.yaml with identical settings — cleaner syntax, native comments, and easier to read. Every field documented here.

View full example → YAML vs JSON →

YAML vs JSON — Why Use config.yaml?

Both config.yaml and openclaw.json support identical fields and functionality. Choose based on your preference:

Why YAML?

  • Native comments with #
  • No trailing comma issues
  • More readable nested structure
  • Multi-line strings without escaping
  • Cleaner, less verbose syntax

Why JSON?

  • Stricter validation
  • No indentation sensitivity
  • Better IDE tooling support
  • Easier to parse programmatically
  • Industry standard for configs

Key tip: OpenClaw auto-detects your format based on file extension. YAML is indentation-sensitive — use 2 spaces, never tabs. Always validate with openclaw config validate before restarting.

Where Is config.yaml?

Default location: ~/.openclaw/config.yaml

Override with environment variable:

export OPENCLAW_CONFIG=/path/to/config.yaml
openclaw start

Quick commands:

  • openclaw config edit — Open config in your default editor
  • openclaw config validate — Validate YAML syntax and required fields
  • openclaw config reload — Reload config without restarting
  • openclaw config show — Print current config (redacts secrets)

LLM Configuration

Configure your language model provider and behavior:

llm:
  provider: anthropic              # "anthropic" | "openai" | "google" | "ollama"
  model: claude-3-5-sonnet         # Model name from selected provider
  apiKey: ${ANTHROPIC_API_KEY}     # Always use env vars for secrets
  baseUrl: ""                      # Custom endpoint (optional, for self-hosted)
  temperature: 0.7                 # 0.0–2.0. Higher = more creative
  maxTokens: 4096                  # Maximum response length
  timeout: 60                      # Seconds
  retries: 3                       # Retry failed requests
  cache: true                      # Prompt caching (reduces cost)
  heartbeatModel: claude-haiku-4   # Lightweight model for health checks
  powerModel: claude-3-5-sonnet    # Complex reasoning tasks
Key Type Default Required Description
provider string — ✓ LLM provider: anthropic, openai, google, or ollama
model string — ✓ Model ID from provider. E.g. claude-3-5-sonnet, gpt-4-turbo
apiKey string — ✓ API key. Use ${ENV_VAR} syntax for environment variables
baseUrl string "" Custom API endpoint. Required for self-hosted or proxy setups
temperature number 0.7 Sampling temperature 0.0–2.0. Higher = more creative, lower = deterministic
maxTokens number 4096 Maximum tokens in response
timeout number 60 Request timeout in seconds
retries number 3 Retry count for failed requests
cache boolean true Enable prompt caching. Reduces token usage and cost
heartbeatModel string — Lightweight model for periodic health checks (saves cost)
powerModel string — Stronger model for complex reasoning. Falls back to model if not set

web.host string 0.0.0.0 Host to bind to. Use 127.0.0.1 for localhost only

Memory Configuration

Configure OpenClaw's persistent memory system:

memory:
  directory: ~/.openclaw/memory     # Where memory files are stored
  memoryFile: memory.json           # Main memory file
  dailyNotes: true                  # Summarize daily activity
  retentionDays: 90                 # Delete old memories
  maxMemoryWords: 10000             # Prune when exceeds limit
  autoSummarise: true               # Auto-compress old memories
  search:
    enabled: true                   # Enable memory search
    fuzzy: true                      # Fuzzy matching
    maxResults: 5                    # Results per search
Key Type Default Required Description
directory string ~/.openclaw/memory Directory where memory files are stored
memoryFile string memory.json Filename for core memory (within directory)
dailyNotes boolean true Create daily summaries of conversations and learnings
retentionDays number 90 Days to keep memory entries. 0 = infinite
maxMemoryWords number 10000 Prune memory when word count exceeds this
autoSummarise boolean true Auto-compress old memories to save tokens
search.enabled boolean true Enable searching memory for past conversations
search.fuzzy boolean true Use fuzzy matching for typos and variations
search.maxResults number 5 Maximum memory entries returned per search

Voice Configuration

Enable speech-to-text and text-to-speech:

voice:
  stt:
    enabled: true
    provider: openai                # "openai" | "google" | "deepgram"
    apiKey: ${OPENAI_API_KEY}
    language: en-US

  tts:
    enabled: true
    provider: openai                # "openai" | "google" | "elevenlabs"
    apiKey: ${OPENAI_API_KEY}
    voiceId: alloy                  # Provider-specific voice
    model: tts-1                    # Model variant

  wakeWord:
    enabled: true
    phrases:
      - "hey openclaw"
      - "openclaw"

  silenceTimeoutMs: 3000            # Stop listening after silence
  interruptOnSpeech: true           # Stop responding when user speaks
  quietHours:
    start: "22:00"
    end: "08:00"
Key Type Default Required Description
stt.enabled boolean false Enable speech-to-text
stt.provider string — ✓* STT provider: openai, google, deepgram
stt.apiKey string — ✓* API key for STT provider
stt.language string en-US Language code (e.g., en-US, fr-FR)
tts.enabled boolean false Enable text-to-speech
tts.provider string — ✓* TTS provider: openai, google, elevenlabs
tts.apiKey string — ✓* API key for TTS provider
tts.voiceId string alloy Voice ID. Provider-specific (e.g., alloy, nova for OpenAI)
tts.model string tts-1 Model variant (e.g., tts-1, tts-1-hd for OpenAI)
wakeWord.enabled boolean false Require wake word before STT activation
wakeWord.phrases array [] List of wake word phrases (case-insensitive)
silenceTimeoutMs number 3000 Milliseconds of silence before stopping audio capture
interruptOnSpeech boolean true Stop TTS when user starts speaking
quietHours.start string — Start of quiet hours (HH:MM format, 24-hour)
quietHours.end string — End of quiet hours (HH:MM format, 24-hour)

Configure periodic health checks and background tasks:

heartbeat:
  enabled: true
  interval: 300                     # Seconds (5 minutes)
  quietHours:
    start: "22:00"
    end: "08:00"
Key Type Default Required Description
enabled boolean true Enable periodic heartbeat/health checks
interval number 300 Interval between heartbeats in seconds
quietHours.start string — Start of quiet hours (HH:MM format, 24-hour). Skips heartbeat during this window
quietHours.end string — End of quiet hours (HH:MM format, 24-hour)

MCP Configuration

Connect Model Context Protocol servers for extended capabilities:

mcp:
  mcpServers:
    brave:
      command: npx
      args:
        - "@modelcontextprotocol/server-brave"
      env:
        BRAVE_API_KEY: ${BRAVE_API_KEY}

    postgres:
      command: /usr/local/bin/mcp-postgres
      args:
        - --database
        - mydb
      env:
        DATABASE_URL: ${DATABASE_URL}
Key Type Default Required Description
mcpServers object {} Map of MCP server names to configuration
mcpServers.{name}.command string — ✓ Command to start the server (e.g., npx, /path/to/binary)
mcpServers.{name}.args array [] Command-line arguments
mcpServers.{name}.env object {} Environment variables. Use ${VAR} syntax for secrets

Logging Configuration

Control OpenClaw's logging behavior:

logging:
  level: info                       # "debug" | "info" | "warn" | "error"
  file: ~/.openclaw/openclaw.log    # Log file path
  maxSizeMb: 100                    # Rotate when file exceeds size
  rotate: true                      # Enable log rotation
Key Type Default Required Description
level string info Log level: debug, info, warn, error
file string ~/.openclaw/openclaw.log Path to log file
maxSizeMb number 100 Max file size (MB) before rotation
rotate boolean true Enable automatic log rotation

YAML Tips & Best Practices

Quote values with special characters

Always quote strings containing colons, hashes, or other special chars: "http://localhost:8000", "#comment"

Use environment variables for secrets

Reference env vars with ${VAR_NAME} syntax. Never hardcode API keys: apiKey: ${ANTHROPIC_API_KEY}

Indentation is critical

Use exactly 2 spaces per indentation level. Never use tabs. Inconsistent indentation causes parse errors.

Validate before restarting

Always run openclaw config validate after editing to catch syntax errors early.
