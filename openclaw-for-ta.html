OpenClaw TA Config

All scripts in this series read from a single ta-config.yaml file. Define your watchlist once, and every tool — the scanner, the alerts, the chart generator — pulls from the same source.

YAML 
# ta-config.yaml — OpenClaw Technical Analysis config

watchlist:
  name: "My TA Watchlist"
  tickers:
    - AAPL
    - MSFT
    - GOOGL
    - NVDA
    - SPY
    - QQQ
    - IWM

signals:
  rsi_oversold:    30      # alert when RSI drops below this
  rsi_overbought:  70      # alert when RSI rises above this
  macd_cross:      true    # alert on MACD bull/bear crosses
  golden_cross:    true    # alert on SMA50/SMA200 crossovers
  bb_squeeze:      true    # alert on Bollinger Band squeeze

alerts:
  email: "you@example.com"
  daily_digest: true       # send a rolled-up digest at market close
  cooldown_hours: 4        # minimum hours between same signal repeating

scanner:
  run_time: "16:15"        # run after market close (Mon–Fri)
  lookback_period: "1y"    # historical data period for indicators
  export_csv: true         # save daily results to scanner_YYYYMMDD.csv

charts:
  enabled: true
  format: "html"           # html (interactive Plotly) or png (matplotlib)
  output_dir: "charts/"
  tickers: ["AAPL","MSFT","SPY"]  # subset for chart generation

Frequently Asked Questions

Is yfinance data reliable enough for signals?
For end-of-day signals and backtesting, yes. Yahoo Finance OHLCV data is generally accurate for daily timeframes. For intraday or tick data you'll need a paid source.
Does pandas-ta work with crypto data?
Yes. Any OHLCV DataFrame works — yfinance can pull crypto tickers like BTC-USD directly.
Can I automate actual trades with this?
OpenClaw is a signal and monitoring framework, not an execution engine. These scripts generate signals; acting on them requires a separate brokerage integration.
How many stocks can the scanner handle?
The S&P 500 with daily candles runs in under 2 minutes on a standard laptop. For larger universes, add caching to avoid re-fetching unchanged data.
