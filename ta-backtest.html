Read This Before You Backtest

Survivorship Bias: yfinance only has data for companies that still exist. Strategies tested on "current S&P 500" miss all the stocks that got delisted. If you backtest on a list that's been cleaned of losers, your results are too good.
Overfitting: If you tweak parameters until results look great, you've built a history-memorizer, not a strategy. This is why parameter sweeps feel seductive but are dangerous. Use them to understand sensitivity, not to pick your "best" parameters.
Transaction Costs: Real trades have commissions, slippage, and spreads. This backtester ignores them — subtract 0.1–0.3% per trade for realism. A strategy showing 1% avg return per trade barely beats costs.
Look-Ahead Bias: Never use future data in your entry/exit signals. The code below uses .shift(1) correctly to avoid this — signals are calculated on yesterday's close, executed on today's open.
What Backtesting IS Good For: Eliminating clearly bad ideas quickly, comparing two approaches objectively, understanding drawdown risk in a strategy, building intuition about indicator behavior.

Core Backtester — RSI Mean Reversion

Here's a complete backtester for a simple RSI-based strategy. It uses next-day open prices to avoid look-ahead bias and tracks win rate, drawdown, and hold time.

Strategy rules:

  • BUY when RSI closes below a threshold (e.g., 30 = oversold)
  • SELL when RSI closes above a threshold (e.g., 60 = recovering) OR after a max hold period (default 30 days)
  • One position at a time — no pyramiding
  • Entry and exit at next-day open (not on the signal bar)
backtest.py 
# pip install yfinance pandas-ta pandas
import yfinance as yf
import pandas_ta as ta
import pandas as pd
from datetime import datetime

def backtest_rsi_mean_reversion(
    ticker: str,
    period: str = "5y",
    rsi_buy: float = 30,
    rsi_sell: float = 60,
    holding_limit_days: int = 30,
) -> dict:
    """
    Strategy:
      BUY when RSI closes below rsi_buy (oversold)
      SELL when RSI closes above rsi_sell OR after holding_limit_days days
      One position at a time (no pyramiding)

    Uses next-day open as entry/exit price to avoid look-ahead bias.
    """
    df = yf.download(ticker, period=period, interval="1d", progress=False)
    df.columns = [c.lower() for c in df.columns]
    df = df[["open","high","low","close","volume"]].dropna()
    df["rsi"] = ta.rsi(df["close"], 14)
    df = df.dropna()

    trades = []
    in_trade = False
    entry_price = None
    entry_date = None
    entry_idx = None

    for i in range(1, len(df) - 1):
        row = df.iloc[i]
        prev_rsi = df.iloc[i - 1]["rsi"]

        if not in_trade:
            # Entry signal on today's close, execute at tomorrow's open
            if prev_rsi < rsi_buy:
                entry_price = df.iloc[i + 1]["open"]  # next-day open
                entry_date  = df.index[i + 1]
                entry_idx   = i + 1
                in_trade = True

        else:
            days_held = i - entry_idx
            # Exit: RSI overbought OR max hold period
            if prev_rsi > rsi_sell or days_held >= holding_limit_days:
                exit_price = df.iloc[i + 1]["open"]
                exit_date  = df.index[i + 1]
                ret = (exit_price - entry_price) / entry_price * 100
                trades.append({
                    "entry_date":  entry_date.date(),
                    "exit_date":   exit_date.date(),
                    "entry_price": entry_price,
                    "exit_price":  exit_price,
                    "return_pct":  ret,
                    "days_held":   days_held,
                    "win":         ret > 0,
                })
                in_trade = False

    return {"ticker": ticker, "trades": trades, "params": {
        "rsi_buy": rsi_buy, "rsi_sell": rsi_sell,
        "holding_limit_days": holding_limit_days, "period": period,
    }}

Analyse Results

Once you've generated trades, this function computes the key performance metrics and prints a readable report with the last 10 trades.

analyse.py 
def analyse_backtest(result: dict) -> None:
    """Print a full backtest report."""
    trades = result["trades"]
    ticker = result["ticker"]
    params = result["params"]

    if not trades:
        print(f"No trades generated for {ticker} with these parameters.")
        return

    returns = [t["return_pct"] for t in trades]
    wins    = [r for r in returns if r > 0]
    losses  = [r for r in returns if r <= 0]

    win_rate   = len(wins) / len(returns) * 100
    avg_return = sum(returns) / len(returns)
    avg_win    = sum(wins) / len(wins) if wins else 0
    avg_loss   = sum(losses) / len(losses) if losses else 0
    best_trade = max(returns)
    worst_trade= min(returns)

    # Max drawdown (peak-to-trough on cumulative returns)
    cum = 0
    peak = 0
    max_dd = 0
    for r in returns:
        cum += r
        if cum > peak:
            peak = cum
        dd = peak - cum
        if dd > max_dd:
            max_dd = dd

    avg_days = sum(t["days_held"] for t in trades) / len(trades)

    print(f"\n{'═'*55}")
    print(f"  Backtest: {ticker} | RSI({params['rsi_buy']}/{params['rsi_sell']}) | {params['period']}")
    print(f"{'═'*55}")
    print(f"  Total trades:    {len(trades)}")
    print(f"  Win rate:        {win_rate:.1f}%")
    print(f"  Avg return/trade:{avg_return:+.2f}%")
    print(f"  Avg win:         {avg_win:+.2f}%")
    print(f"  Avg loss:        {avg_loss:+.2f}%")
    print(f"  Best trade:      {best_trade:+.2f}%")
    print(f"  Worst trade:     {worst_trade:+.2f}%")
    print(f"  Max drawdown:    -{max_dd:.2f}%")
    print(f"  Avg days held:   {avg_days:.1f}")
    print(f"{'─'*55}")

    # Last 10 trades
    print(f"\n  Last 10 trades:")
    print(f"  {'Entry':<12} {'Exit':<12} {'Entry $':>9} {'Exit $':>9} {'Return':>8} {'Days':>5}")
    for t in trades[-10:]:
        arrow = "✓" if t["win"] else "✗"
        print(f"  {str(t['entry_date']):<12} {str(t['exit_date']):<12} "
              f"${t['entry_price']:>8.2f} ${t['exit_price']:>8.2f} "
              f"{t['return_pct']:>+7.2f}% {t['days_held']:>5}  {arrow}")

result = backtest_rsi_mean_reversion("AAPL", period="5y", rsi_buy=30, rsi_sell=60)
analyse_backtest(result)

Parameter Sweep — Find the Best RSI Thresholds

Use a parameter sweep to understand how sensitive your strategy is to different RSI buy/sell levels. ⚠️ WARNING: Choosing the "best" parameters from this table is overfitting. Use it for understanding sensitivity, not picking parameters.

parameter_sweep.py 
def rsi_parameter_sweep(ticker: str, period: str = "5y") -> pd.DataFrame:
    """
    Test multiple RSI buy/sell combinations and compare results.
    WARNING: choosing the 'best' params from this table is overfitting.
    Use it for understanding sensitivity, not picking parameters.
    """
    buy_levels  = [20, 25, 30, 35]
    sell_levels = [55, 60, 65, 70]

    rows = []
    for buy in buy_levels:
        for sell in sell_levels:
            if sell <= buy + 20:
                continue  # Nonsensical combination
            result = backtest_rsi_mean_reversion(ticker, period, buy, sell)
            trades = result["trades"]
            if not trades:
                continue
            returns = [t["return_pct"] for t in trades]
            rows.append({
                "rsi_buy":    buy,
                "rsi_sell":   sell,
                "n_trades":   len(trades),
                "win_rate":   round(len([r for r in returns if r > 0]) / len(returns) * 100, 1),
                "avg_return": round(sum(returns) / len(returns), 2),
                "total_return": round(sum(returns), 2),
            })

    df = pd.DataFrame(rows).sort_values("avg_return", ascending=False)
    print(f"\n{'═'*60}")
    print(f"  RSI Parameter Sweep: {ticker} | {period}")
    print(f"  ⚠️  Selecting best params from this table = overfitting")
    print(f"{'═'*60}")
    print(df.to_string(index=False))
    return df

sweep = rsi_parameter_sweep("SPY", period="5y")

Don't fall into the overfitting trap: If you pick (30, 65) because it has the highest win rate, you're optimizing the historical data. The best historical parameters rarely remain best on new data. Instead, pick parameters before backtesting (based on theory or convention), then use the sweep to stress-test that choice.

Comparing Buy-and-Hold vs Strategy

A strategy only matters if it beats the alternative — which for most stocks is simply buying and holding. Here's how to compare:

compare.py 
def vs_buy_hold(result: dict, ticker: str, period: str = "5y") -> None:
    """Compare strategy total return vs simple buy-and-hold."""
    df = yf.download(ticker, period=period, interval="1d", progress=False)
    df.columns = [c.lower() for c in df.columns]
    df = df.dropna()

    buy_hold = (df["close"].iloc[-1] - df["close"].iloc[0]) / df["close"].iloc[0] * 100

    trades = result["trades"]
    strategy_total = sum(t["return_pct"] for t in trades)

    print(f"\n  Buy & Hold ({period}): {buy_hold:+.1f}%")
    print(f"  Strategy total:      {strategy_total:+.1f}%  ({len(trades)} trades)")
    print(f"  Note: strategy returns are summed, not compounded.")
    print(f"  Market invested:     {len(trades) * (sum(t['days_held'] for t in trades)/len(trades) if trades else 0):.0f} avg days total.")

vs_buy_hold(result, "AAPL", "5y")

In this case, the RSI strategy beat buy-and-hold. But notice: the strategy was in the market only 768 days total (out of 1825 days). That means it was sitting in cash 60% of the time. So the per-day return is better, even if the total return looks lower.

Common Backtest Mistakes

Here are the most common pitfalls that make backtests look better than they are:

Mistake What Happens How to Avoid
Look-ahead bias Signal uses future data (e.g., using close price before market open) Always use .shift(1) before acting. Signal on yesterday's close, execute on today's open.
Survivorship bias Only testing winners; ignores delisted or bankrupt stocks Use point-in-time universe data. Know which stocks existed when.
Overfitting Tune parameters to history; works great on past data, fails on new data Test on out-of-sample period. Use only standard parameter ranges.
Ignoring costs Returns look better than reality Subtract 0.1–0.3% per trade (commission + slippage).
Small sample 5 trades = huge variance, results are noise Need 30+ trades minimum for meaningful signal.
Data snooping Test until something "works", claim it's a discovery Pre-commit to hypothesis first. Don't mine for p-values.

Key Backtest Metrics

Here's a quick reference for interpreting backtest results:

Metric Good Range Notes
Win Rate 45–65% High win rate alone isn't enough — check avg win vs avg loss. A 70% win rate with 0.5% wins and 2% losses is bad.
Avg Return/Trade > 0.5% After costs (0.2% per trade), must beat 0. If avg return is 0.3%, you're negative after costs.
Max Drawdown < 20% Worst peak-to-trough loss. Tells you the pain during the strategy. Higher = less risk-adjusted.
# Trades > 30 Fewer = statistically meaningless. 100+ trades = better signal.
Sharpe Ratio > 1.0 avg_return / std_dev. Proxy: > 1 means risk-adjusted return is decent.
One important caveat: Backtests are always biased optimistic. Historical data is clean, commissions are small, and you know all past prices. Real trading adds friction. Assume your actual edge is 30–50% of the backtest edge.

Putting It All Together

Here's a complete script that runs a backtest, analyzes it, and compares to buy-and-hold:

full_example.py 
#!/usr/bin/env python3
"""Complete backtest workflow."""

import yfinance as yf
import pandas_ta as ta
import pandas as pd

# [Include backtest_rsi_mean_reversion, analyse_backtest,
#  vs_buy_hold, rsi_parameter_sweep functions above]

if __name__ == "__main__":
    ticker = "AAPL"
    period = "5y"

    print(f"\n🦎 Running backtest for {ticker}...")

    # Run backtest with standard parameters
    result = backtest_rsi_mean_reversion(
        ticker=ticker,
        period=period,
        rsi_buy=30,
        rsi_sell=60,
        holding_limit_days=30
    )

    # Analyse
    analyse_backtest(result)

    # Compare to buy-and-hold
    vs_buy_hold(result, ticker, period)

    # Optional: run parameter sweep
    print(f"\n\n📊 Running parameter sweep...")
    sweep = rsi_parameter_sweep(ticker, period)

    print("\n✓ Backtest complete. Remember:")
    print("  - These are historical results only")
    print("  - Real trading has costs and slippage")
    print("  - Don't overfit to past data")
